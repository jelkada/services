Services and dependency injection (see on github)

  • This example uses two service classes: usersService and counterService (counterService is also injected into usersService).
  • The services share some common functionality for cross component communication.
  • The usersService updates the activeUsers and inactiveUsers[] arrays and refresh the list of users accordingly.
  • The counterService counts the total number of status changes and this.counterService.onIncrement() is invoked from few components.
  • The users components have an “statusChanged” EventEmitter which is emitted each time user status is updated.
  • The app components listens to the (statusChanged)="onStatusChanged() event and updates the counter via string interpolation.

Services highlights:

  • Angular Services centralize common tasks and communicate between component (passing data around)
  • Services provide application data and business logic to components.
  • Services are objects that are wired together using dependency injection (DI).
  • You can inject/use a service in components, giving the components access to that Service functions.
  • The component then has a dependency on the service, which is registered via the component’s provider array.

Usage guidelines:

  • Create a simple class that will be used as a Service, such as "UsersService" (in users.service.ts)
  • Set up the data and common functions that the Service provides
  • Use the @Injectable() decorator to indicate if the Service includes other services or dependencies
  • Import the Service in your module component.
    Ex: import { UsersService } from './users.service';
  • Add the Service to the providers array in your module or component.
    Ex: providers: [UsersService]
  • Include the Service in the component or service constructor, where the service is used.
    Ex: constructor(private usersService: UsersService) {}
